# DNN-Opt v0.8.0 — Phase 6B: 优化增强 (2026-04-09)

## 版本概述

v0.8.0 版本实现了三个关键优化方向：
1. **小 shape GEMM 性能优化** — 预取 + 软件流水线 + 激进循环展开
2. **多线程优化增强** — NUMA-aware, False sharing 预防, big.LITTLE 调度
3. **SME 前瞻优化框架** — FMOPA/BFMOPA/SMOPA 指令支持

## 新增功能

### 1. 小 shape GEMM 优化 (`gemm_smallm_fp32_v2.cpp`)

**优化技术：**
- PRFM 预取指令 (L1/L2 cache)
- 8x K-unrolling (vs 原来 4x)
- 软件流水线 (load N+8 while computing N)
- 更好的寄存器分配

**性能提升预期：**
- 小 shape (M,N≤16): 1.2-1.5x 加速
- GEMV (M=1 或 N=1): 1.3-1.6x 加速

### 2. 多线程优化增强 (`gemm_threading_v2.cpp`)

**新增功能：**
- big.LITTLE 感知线程选择
- NUMA-aware 内存分配框架
- False sharing 预防 (cache line padding)
- Fuzzy decomposition for load balancing
- Work distribution 策略选择 (static/dynamic/guided)

**线程策略：**
```
问题大小              使用的核心
--------------------------------------
< 100K FLOPs/core     1 thread
1M-10M FLOPs/core     仅 big cores
> 10M FLOPs/core      big + little cores
```

### 3. SME 前瞻优化 (`gemm_sme.cpp`)

**框架特性：**
- SME 能力检测 (运行时)
- ZA tile 管理
- FMOPA/BFMOPA/SMOPA 指令包装
- Streaming Mode 管理
- SME2 2-vector/4-vector 操作

**注意：** 当前 Neoverse N2 不支持 SME，此代码为未来硬件准备。

## 使用方法

### 编译
```bash
cd /root/onednn-arm-opt/build
cmake .. -DCMAKE_BUILD_TYPE=Release
make -j$(nproc)
```

### 运行基准测试
```bash
# 小 shape GEMM 性能测试
./benchmarks/bench_small_gemm_v2

# 多线程性能测试
./benchmarks/bench_threading

# 常规 GEMM 测试
./benchmarks/bench_gemm
```

### 使用优化的小 shape GEMM
```cpp
#include "dnnopt/gemm/gemm.h"

// v2 优化版本会自动被 dispatch
dnnopt::sgemm(M, N, K, alpha, A, lda, B, ldb, beta, C, ldc);
```

## 技术细节

### 预取距离调优
```cpp
// L1 cache prefetch distance
kPrefetchL1Dist = 8 iterations

// L2 cache prefetch distance
kPrefetchL2Dist = 16 iterations
```

### False Sharing 预防
```cpp
struct PaddedCounter {
    alignas(64) int64_t value;  // Cache line padding
};
```

### SME 指令示例
```assembly
// FMOPA: FP32 outer product into ZA tile
fmopa za0.s, p0/m, z0.s, z1.s

// BFMOPA: BF16 outer product
bfmopa za0.s, p0/m, z0.h, z1.h

// SMOPA: INT8 outer product (int32 accumulate)
smopa za0.d, p0/m, z0.b, z1.b
```

## 已知限制

1. **SME 代码**: 当前仅软件 fallback，需要 SME 硬件和编译器支持
2. **NUMA**: 框架已就绪，但 libnuma 集成待完成
3. **Thread affinity**: 仅检测到 big.LITTLE，实际亲和性设置待实现
